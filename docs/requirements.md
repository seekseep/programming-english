# 要件定義

## 概要

プログラミングで使う英語を学習するブラウザゲーム。
ステージをクリアしてポイントを稼ぎ、アバターを育てて着せ替えを楽しむ。

## 技術スタック

- フロントエンド: TanStack Start + React + Tailwind CSS
- データ永続化: localStorage（サーバー不要）
- デプロイ: Cloudflare Pages
- アセット作成ツール: scripts/ 配下に配置

## 機能一覧

### 1. 単語データ

プログラミングで頻出する英単語・フレーズのデータセット。

- 単語、意味（日本語）、例文、カテゴリを持つ
- JSON ファイルとして管理し、ビルド時にバンドルする

#### カテゴリ案

| 変数・型     | variable, string, boolean, integer, null        |
| 制御構文     | if, else, loop, while, return                   |

| データ構造   | array, object, map, set, queue, stack           |
| 開発ツール   | commit, branch, merge, deploy, debug            |
| エラー・例外 | error, exception, throw, catch, undefined       |

### 2. ゲーム（ステージ制クイズ）

ステージごとに異なるカテゴリ・難易度の問題を出題する。

#### 出題形式

##### 4択モード

- 英単語が表示され、4つの日本語の選択肢から正しい意味を選ぶ
- 最も基本的なモード

##### ペア選びモード

- 画面の左側に英単語、右側に日本語の意味が複数表示される
- プレイヤーは左右から正しいペアを選んでマッチさせる
- 全ペアを正しくマッチさせたらクリア

##### 神経衰弱モード

- カードが裏返しの状態で並んでいる
- 2枚めくって英単語と日本語の意味がペアになっていれば正解
- 全ペアを見つけたらクリア

##### ゆっくり英語表示モード（4択モードの拡張版）

- 英単語が1文字ずつゆっくり表示される
- 表示中に、その意味に合う日本語の選択肢があれば素早く選ぶ
- 少ない文字数で正解するほど高スコア（早く気づけるほど高得点）

#### ステージ構成

- 全8ステージ（1カテゴリ = 1ステージ）
- 1ステージ = 20単語
- 各ステージで4モード全てプレイ可能
- 1プレイ = 10単語（20単語の中からランダムに選出）
- 単語にはモード専用のものと全モード共通のものがある
  - 全単語を正解するには全モードをプレイする必要がある
- クリア判定はステージ単位（モード横断で80%＝16単語以上正解で次ステージ解放）
- 時間制限なし。ただし回答時間に応じてボーナスポイント（早いほど高得点）
- 間違えた単語は1回のプレイ内で正解するまで再出題される
- 正答率に応じてスコア・ポイントを付与

#### 決定済み事項

- [x] 出題形式 → 4択 / ペア選び / 神経衰弱 / ゆっくり英語表示 の4種
- [x] 1ステージの単語数 → 20単語
- [x] 1プレイの出題数 → 10単語
- [x] クリア条件 → ステージ単位で80%正解（16/20）、モード横断
- [x] ステージのアンロック条件 → 80%正解で次ステージ解放
- [x] 時間制限 → なし。回答速度でボーナスポイント
- [x] 再出題 → 間違えた単語は同プレイ内で正解するまで再出題
- [x] ステージ数 → 8（カテゴリと1対1）

### 3. 単語帳

学習した単語を閲覧・復習するための機能。

- 全単語を一覧表示
- 未学習の単語は「？」で表示（単語を集めるコレクション要素）
- 正答数に応じて習熟度を表示
- カテゴリでフィルタリング
- 復習モードは設けない（ゲームモードで繰り返しプレイする）

#### 習熟度

| 習熟度   | 条件                  |
| -------- | --------------------- |
| 未学習   | 正答0回（「？」表示） |
| 見習い   | 正答1〜2回            |
| 理解     | 正答3〜5回            |
| 習得     | 正答6〜9回            |
| マスター | 正答10回以上          |

#### 決定済み事項

- [x] 未学習単語 → 「？」で表示、コレクション要素
- [x] 習熟度 → 正答数に応じて段階表示
- [x] 復習モード → なし

### 4. アバターシステム

#### キャラクター

- ロボット型キャラクター
- 背景透過PNGをパーツごとに用意し、重ね合わせて表示
- 画像はOpenAI APIで生成

#### 表示場所

- ホーム画面に大きく表示
- クイズ中にも応援キャラとして表示

#### 成長段階

ステージクリア数に応じて成長し、装備可能なスロットが増える。

| 段階 | アイテムスロット数 | アンロック条件    |
| ---- | ------------------ | ----------------- |
| 1    | 1                  | 初期状態          |
| 2    | 2                  | 2ステージクリア   |
| 3    | 3                  | 4ステージクリア   |
| 4    | 4                  | 6ステージクリア   |
| 5    | 5                  | 8ステージ全クリア |

#### アイテムスロット

| スロット     | 解放段階 | 例                                 |
| ------------ | -------- | ---------------------------------- |
| 頭部パーツ   | 1        | アンテナ、バイザー、ヘッドライト   |
| ボディパーツ | 2        | アーマー、カラーリング、ロゴ       |
| アームパーツ | 3        | ドリル腕、レーザー腕、ツールハンド |
| バックパーツ | 4        | ジェットパック、翼、ソーラーパネル |
| エフェクト   | 5        | 電流エフェクト、オーラ、コード背景 |

#### ポイントとアイテム

- ステージクリア時にスコアに応じたポイントを獲得
- ポイントを消費してストアでアイテムを購入
- 成長段階はポイントとは無関係（ステージクリア数で解放）

#### 決定済み事項

- [x] キャラクター → ロボット型
- [x] イラスト方式 → 背景透過PNG + OpenAI API生成
- [x] アイテムスロット → 頭部 / ボディ / アーム / バック / エフェクト の5種
- [x] 成長段階の解放条件 → ステージクリア数
- [x] 表示場所 → ホーム画面（大）+ クイズ中（応援キャラ）

### 5. アイテムストア

ポイントを使ってアバターのアイテムを購入する。

- スロット別にアイテムを表示
- 購入済み / 未購入を区別
- ポイント残高を表示
- レアリティやセールの仕組みは設けない

#### ポイントバランス

| 項目                 | ポイント                    |
| -------------------- | --------------------------- |
| 1プレイ基本報酬      | 50〜100pt（正答率に応じて） |
| 速度ボーナス         | 最大+50pt                   |
| アイテム価格（安い） | 100pt                       |
| アイテム価格（中間） | 300pt                       |
| アイテム価格（高い） | 500pt                       |

#### アイテム総数

| スロット     | アイテム数 |
| ------------ | ---------- |
| 頭部パーツ   | 5個        |
| ボディパーツ | 5個        |
| アームパーツ | 5個        |
| バックパーツ | 5個        |
| エフェクト   | 5個        |
| **合計**     | **25個**   |

※ アイテム画像は全成長段階で共通（段階ごとの差分なし）

#### 決定済み事項

- [x] ポイントバランス → 1プレイ50〜150pt、アイテム100〜500pt
- [x] アイテム総数 → 各スロット5個、計25個
- [x] レアリティ → なし
- [x] セール → なし

### 6. アセット作成ツール（scripts/）

2段階のパイプラインでアセットを作成する。

#### ツール1: アイテムデータ生成ツール

- アイテムのメタデータ（名前、スロット、価格、説明）をJSONとして生成
- このデータが先にできる

#### ツール2: 画像生成ツール

- アイテムデータをもとにOpenAI APIで背景透過PNGを生成
- 品質を揃えるための共通プロンプトテンプレートを用意
- スロットごとにプロンプトを調整

#### 画像合成方式: フルキャンバスオーバーレイ

- 全画像を同じキャンバスサイズ（例: 512×512）で統一
- ベース画像もアイテム画像も同サイズの背景透過PNG
- アイテム画像は「ベース画像上の正しい位置にパーツが描かれた透過PNG」
- コンポーネント側はレイヤーを重ねるだけ（位置計算不要）
- 成長段階ごとに体型が変わるため、アイテムも段階ごとに用意

#### レイヤー順序（z-index）

| z-index | レイヤー                       |
| ------- | ------------------------------ |
| 5       | エフェクト                     |
| 4       | 頭部パーツ                     |
| 3       | アームパーツ                   |
| 2       | ボディパーツ                   |
| 1       | ベースロボット                 |
| 0       | バックパーツ（ロボットの後ろ） |

#### 必要画像数

| 種類                                    | 枚数     |
| --------------------------------------- | -------- |
| ベースロボット（3種類 × 5段階）         | 15枚     |
| アイテム（5スロット × 各5個、段階共通） | 25枚     |
| **合計**                                | **40枚** |

- ベースロボットは3種類から選べる
- 成長段階ごとに体型は変わるが、アイテム画像は全段階共通で使い回す

#### 共通プロンプト方針

画像の品質・スタイルを統一するために以下を固定する:

- キャラクター: ロボット
- 画像形式: 背景透過PNG
- キャンバスサイズ: 512×512
- 画風: イラスト調
- 構図: 正面向き、キャンバス内の固定位置に描写
- アイテム画像はベースロボットと同じ位置関係で描写

#### 決定済み事項

- [x] 画風 → イラスト調
- [x] キャンバスサイズ → 512×512
- [ ] プロンプトの具体的な内容（アイテムデータ確定後に詰める）

## データ構造（案）

### localStorage に保存するデータ

```typescript
type GameData = {
  // プレイヤー情報
  player: {
    totalPoints: number // 累計獲得ポイント
    currentPoints: number // 使用可能ポイント
    avatarLevel: 1 | 2 | 3 | 4 | 5
    equippedItems: string[] // 装備中のアイテムID
    ownedItems: string[] // 所持アイテムID一覧
  }

  // ステージ進行
  stages: Record<
    string,
    {
      cleared: boolean
      bestScore: number
      attempts: number
    }
  >

  // 単語学習履歴
  wordHistory: Record<
    string,
    {
      correctCount: number
      incorrectCount: number
      lastStudied: string // ISO日付
    }
  >
}
```
